Keywords for Variable Declaration

Function Scoped Variables

Function Scoped variables are the most common type of variables

This is the type of variable that you probably already know

These variables are available anywhere inside the function where they were declared in

var

Any variable declared using the var keyword is going to be Function Scoped

Variables declared with var are hoisted

Example 1:

Displaying variables on the console

                                                
function myExample() {
    if (true) {
        var myVar = "Hello World!"
    }
    console.log(myVar)
}
myExample()	

console.log(myVar)
Console: 
                                                
Hello World!
error

In this example we were able to access myVar from outside the if statement and successfully log it on the console.

However, we were NOT able to access it from outside the function itself

Example 2:

Hoisting

                                        
console.log(myVar)
var myVar = 'true'
console.log(myVar)
Console: 
                                        
undefined
true

With this example we show hoisting makes it possible to access the variable before it was even declared.

The value remains undefined until we reach the actual line were the variable was written.

All variables declared with var will be 'pushed' to the top of the block were they were declared.

The variables themselves will be available from the start of the block, but the values we have given them will only be available after the original line of declaration.

Variables that use this keyword can be declared multiple times

Block Scoped Variables

Variables declared with the use of let and const are Block Scoped Variables

Block Scoped variables:

  • Are not hoisted and cannot be accessed before their initialization.
  • Are only available inside of the block where they were declared.

What are blocks?

  • Blocks are structures inside of our code, like the if statements and loops.
Example: Displaying variables on the console 
                                            
                                                function myExample() {
if (true) {
    let myLet = "Hello World!"
    const myConst = "Hello Again!"

    console.log(myLet)
    console.log(myConst)
}

    console.log(myLet)		
    console.log(myConst)
}
myExample()
Console: 
                                            
                                                Hello World!
                                                Hello Again!
                                                
                                                error
                                                error

We successfully displayed myVar and myLet when we called them from inside the block were they were declared.

We were not able to display them when we tried to reach from outside the if statement.

let

let is Block Scoped

Differently from var, variables declared with let cannot be initialized more than one time.

let variables can be reassigned however.

const

This keyword is block scoped

The const keyword is used for defining constant values

you will not be able to change it's value later on

const variables can only be initialized once

Special Case

When we use const to define Objects, Arrays or Methods we can still "change" values.

Actually we can't change the "keys" we have declared, but we can reassign the "values" that these "keys" hold.

We could say that the const keyword defines a "constant blueprint" for the Object, Array or Method, where the "keys" are constant, but the values that they can have are allowed to change.

Example: Using const 
                                                    
const myConst = {
    name:"John",
    age:32,
    married: false
}

myConst.name = "Michael"
myConst.married = true
myConst.lastName = "Stevens"

console.log(myConst.name)
console.log(myConst.married)
console.log(myConst.lastName)
Console: 
                                                    
Michael
true
error

We successfully altered and displayed the .name and .married properties.

We were not able to display .lastName
